Overview

I build custom poker OCR and poker automation systems that turn table visuals into structured, actionable data—plus optional Poker BOT execution and poker AI / solver-connected decision paths for permitted tooling. The pipeline starts with Poker Eye, which extracts cards, stacks, bets, and UI states directly from the screen using OCR and image processing. Poker Reader then converts this raw data into complete hand histories by tracking sequences, applying game logic, and handling missing or delayed information.

From there, the system connects to solver logic through APIs, local datasets, live scraping, or AI models to generate strategy outputs. When needed, a Poker BOT layer executes decisions by interacting with the interface in real time.

Each component can run independently or as a full pipeline, supporting both live and offline use cases—built for real environments, handling complex layouts, animations, and inconsistencies without relying on direct platform access.
